1.用户登录
后端：整体思路
参数判断  用户姓名  非空
        密码  非空
        2. 通过用户姓名查询 用户 返回用户对象
        判断用户对象是否为空
        用户对象不为空  判断数据库中的密码 与前台密码 作比较 判断密码是否正确
        密码不正确 则登陆失败 密码正确 登录成功
    controller  控制层  接收请求 响应结果
    1. 通过形参接收客户端参数
    2.调用业务逻辑层 的登录方法 得到登录结果
    3.响应数据给客户端
    service     业务逻辑层  做判断  ：非空判断 条件判断
    1.参数判断 判断用户姓名密码非空
    如果参数为空 抛异常  异常被控制层捕获
    不为空  调用数据访问层 通过用户名查询用户记录 返回用户对象
    如果对象为空  则抛异常 异常被控制层捕获
    判断 密码是否正确 比较客户端密码与数据库密码
    密码不正确抛异常
    密码正确 登陆成功
    dao层        数据访问层  数据库crud

2. 修改密码
   后端实现
   controller 层
   1. 通过形参 接收前端的参数 (原始密码 新密码 确认密码 )
   2.  通过request 对象获取cookie 中的用户id
   3. 调用service层 修改密码的功能  得到ResultInfo 对象
   4. 返回ResultInfo 对象
   service 层
   1.  接收四个参数 用户id  原密码 新密码 确认密码
   2. 通过用户id查询用户 返回用户对象
   3. 参数校验 用户对象是否存在 （用户对象是否为空)   判断原始密码是否为空 判断原始密码是否正确  判断系密码 是否与 原始密码一致 (不允许一致 )
   判断重复密码是否为空 判断确认密码是否与新密码一致
   4. 设置新密码  将新密码 加密
   5. 执行更新操作 判断受影响的 行数
   mapper层
   1. 根据用户ID 修改用户密码
